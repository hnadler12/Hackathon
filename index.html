What is BADHAT?

"BADHAT's philosophy is based on absolute emergence: the application's structure arises from execution, without any pre-existing doctrine. The tool imposes nothing: no controllers, no dependency containers, no complex middleware, no heavy configuration, no pre-established architecture. The HTTP request is the only reality, and the file system is the only guide."

Quickstart

Get BADHAT running in five minutes

1. Initialize Project

mkdir myproject && cd myproject
git init
mkdir -p app/io/{route,render} public
touch .gitignore

2. Add BADHAT Core via Subtree

git fetch --depth=1 git@github.com:lareponse/BADHAT.git main
git subtree add --prefix=add/badhat FETCH_HEAD --squash
  • --squash garde BADHAT comme un seul commit
  • --depth=1 garde le fetch léger

3. Create Entry Point (Canonical)

<?php
// public/index.php — BADHAT Entry Point
set_include_path(__DIR__ . '/..' . PATH_SEPARATOR . get_include_path());
$install = require 'add/badhat/trap.php';
require 'add/badhat/map.php';
require 'add/badhat/run.php';
require 'add/badhat/http.php';
require 'add/badhat/pdo.php';
use function bad\map\{hook, seek, look};
use function bad\http\{headers, out};
use function bad\run\loot;
use function bad\pdo\db;
use const bad\trap\HND_ALL;
use const bad\map\REBASE;
use const bad\run\{INVOKE, BUFFER, RESULT};
use const bad\http\ONE;

// Bootstrap
$install(HND_ALL);

// DB bootstrap (REQUIRED before qp())
$pdo = new PDO(
    getenv('DB_DSN_')  ?: 'sqlite:' . __DIR__ . '/../db.sqlite',
    getenv('DB_USER_') ?: null,
    getenv('DB_PASS_') ?: null,
    [
        PDO::ATTR_ERRMODE            => PDO::ERRMODE_EXCEPTION,
        PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
    ]
);
db($pdo);

// Normalize request path
$io   = __DIR__ . '/../app/io';
$base = realpath($io . '/route') . '/';
$key  = hook($_SERVER['REQUEST_URI'], "\0");

// Phase 1 — Route (logic)
$loot = [];
$route = seek($base, $key, '.php');
if ($route) {
    [$file, $args] = $route;
    $loot = loot([$file], $args, INVOKE);
}

// Phase 2 — Render (presentation)
$render = look($io . '/render/', $key, '.php', REBASE);
if ($render) {
    $loot = loot([$render], $loot, BUFFER | INVOKE);
}

// Output
if (isset($loot[RESULT]) && is_string($loot[RESULT])) {
    headers(ONE, 'Content-Type', 'text/html; charset=utf-8');
    exit(out(200, $loot[RESULT]));
}
exit(out(404, 'Not Found'));

Home/Docs

Docs

map.php

The atelier begins here. map.php hooks the URL to reality, seeks in the depth of folders the piece that must respond, and lets you decide what happens.
Each hook, seek, and look is a conscious, traceable gesture shaping the request. It reminds you that every route has a life and a traceable cause.

run.php

run.php orchestrates the gesture: include, invoke, buffer. It does no magic, it executes exactly what you choose.
loot() and flags are your brush and palette: you define the motion and the outcome. Precision and intention are everything here.

http.php

http.php translates your gestures into tangible output: headers, out, csp_nonce. Here, the flow becomes visible, each request manifest.
Flags are the compass of your HTTP writing. Control is not hidden; it is explicit, immediate, and measurable.

trap.php

trap.php sets the frame for safety and observability: errors, exceptions, shutdowns — each incident is accountable.
Like an artist tracing their process, every event has a cause, every log a reference. Chaos is contained, visible, and interpretable.

pdo.php

pdo.php is your connection to the material: the database. db(), qp(), trans() give you mastery over interactions,
yet no illusions: you see errors, you choose how to handle them. Your data has structure, and structure is respected.

auth.php

auth.php is minimal access control. checkin() and checkout() are keys you hand over and reclaim. Security is not decorative;
it is explicit, observable, and actionable. Every login is deliberate, every logout accountable.

csrf.php

csrf.php manages the lifecycle and integrity of tokens. Every token reminds you that form has a cycle and actions are controlled.
TTL and validation define the visible limits of your workspace. It enforces discipline without obstructing creativity.

rfc.php

rfc.php enforces rigor: each field name, value, and URL path adheres strictly to standards. Reality allows no arbitrary interpretation;
each request is a sculpture constrained by its framework. Standards are the invisible scaffolding of your expressive work.

Examples

Content Management (Blog Engine)

<?php
// app/io/route/posts/edit.php
use function bad\auth\checkin;
use function bad\csrf\csrf;
use function bad\pdo\qp;
use function bad\http\{headers, out};
use const bad\csrf\CHECK;
use const bad\http\ONE;

return function($args) {
    if (!checkin()) {
        headers(ONE, 'Location', '/login');
        exit(out(302));
    }

    $id = $args[0] ?? throw new \InvalidArgumentException('Post ID required');

    if ($_POST) {
        csrf(CHECK, '_csrf') || exit(out(403, 'Invalid token'));
        qp("UPDATE posts SET title=?, body=? WHERE id=?",
            [$_POST['title'], $_POST['body'], $id]);
        headers(ONE, 'Location', "/posts/$id");
        exit(out(302));
    }

    return ['post' => qp("SELECT * FROM posts WHERE id = ?", [$id])->fetch()];
};

E-commerce Backend

<?php
// app/io/route/admin/orders.php
use function bad\auth\checkin;
use function bad\pdo\qp;
use function bad\http\{headers, out};
use const bad\http\ONE;

return function($args) {
    if (!checkin()) {
        headers(ONE, 'Location', '/login');
        exit(out(302));
    }

    $status = $_GET['status'] ?? 'pending';
    $orders = qp("
        SELECT o.*, u.email, COUNT(oi.id) as items
        FROM orders o
        JOIN users u ON o.user_id = u.id
        LEFT JOIN order_items oi ON o.id = oi.order_id
        WHERE o.status = ?
        GROUP BY o.id
        ORDER BY o.created_at DESC
    ", [$status])->fetchAll();

    return compact('orders', 'status');
};

JSON API

<?php
// app/io/route/api/users.php
use function bad\pdo\qp;
use function bad\http\{headers, out};
use const bad\http\ONE;

return function($args) {
    $action = $args[0] ?? 'list';
    $id = $args[1] ?? null;

    $data = match($action) {
        'list' => qp("SELECT id, name FROM users")->fetchAll(),
        'get'  => qp("SELECT * FROM users WHERE id = ?", [$id])->fetch(),
        default => throw new \InvalidArgumentException('Unknown action')
    };

    headers(ONE, 'Content-Type', 'application/json');
    exit(out(200, json_encode($data)));
};

Dashboard Metrics

<?php
// app/io/route/dashboard/metrics.php
use function bad\pdo\qp;

return function($args) {
    $range = $args[0] ?? '24h';
    $hours = $range === '24h' ? 24 : 168;

    return [
        'metrics' => qp("
            SELECT
                DATE_FORMAT(timestamp, '%H:00') as hour,
                SUM(requests) as total,
                AVG(response_time) as avg_time
            FROM metrics
            WHERE timestamp >= DATE_SUB(NOW(), INTERVAL ? HOUR)
            GROUP BY hour
        ", [$hours])->fetchAll(),
        'range' => $range
    ];
};

File Upload

<?php
// app/io/route/files/upload.php
use function bad\auth\checkin;
use function bad\csrf\csrf;
use function bad\pdo\qp;
use function bad\http\{headers, out};
use const bad\csrf\CHECK;
use const bad\http\ONE;

return function($args) {
    if (!checkin()) {
        headers(ONE, 'Location', '/login');
        exit(out(302));
    }

    if (!$_FILES)
        return ['max_size' => ini_get('upload_max_filesize')];

    csrf(CHECK, '_csrf') || exit(out(403, 'Invalid token'));

    $file = $_FILES['file'];
    $path = 'uploads/' . date('Y/m/') . uniqid() . '_' . basename($file['name']);

    is_dir(dirname($path)) || mkdir(dirname($path), 0755, true);
    move_uploaded_file($file['tmp_name'], $path);

    qp("INSERT INTO files (name, path, size, user_id) VALUES (?, ?, ?, ?)",
        [$file['name'], $path, $file['size'], checkin()]);

    headers(ONE, 'Location', '/files');
    exit(out(302));
};

IoT Data Collector

<?php
// app/io/route/api/sensors.php
use function bad\pdo\qp;
use function bad\http\{headers, out};
use const bad\http\ONE;

return function($args) {
    headers(ONE, 'Content-Type', 'application/json');

    $device_id = $args[0] ?? exit(out(400, '{"error":"Device ID required"}'));

    $data = json_decode(file_get_contents('php://input'), true);

    $stmt = qp("INSERT INTO readings (device_id, type, value, ts) VALUES (?, ?, ?, ?)", []);

    $count = 0;
    foreach ($data['sensors'] as $s) {
        $stmt->execute([$device_id, $s['type'], $s['value'], $s['ts']]);
        ++$count;
    }

    exit(out(200, json_encode(['ok' => true, 'count' => $count])));
};

Multi-tenant SaaS

<?php
// app/io/route/app/projects.php
use function bad\pdo\{db, qp};

return function($args) {
    $tenant = resolve_tenant($_SERVER['HTTP_HOST']);

    db(new \PDO(
        "mysql:host=localhost;dbname=tenant_{$tenant['id']};charset=utf8mb4",
        getenv('DB_USER'),
        getenv('DB_PASS'),
        [\PDO::ATTR_ERRMODE => \PDO::ERRMODE_EXCEPTION]
    ));

    return [
        'projects' => qp("
            SELECT p.*, COUNT(t.id) as tasks
            FROM projects p
            LEFT JOIN tasks t ON p.id = t.project_id
            GROUP BY p.id
        ")->fetchAll(),
        'tenant' => $tenant
    ];
};

Transaction Example

<?php
// app/io/route/trading/execute.php
use function bad\auth\checkin;
use function bad\csrf\csrf;
use function bad\pdo\{qp, trans};
use function bad\http\{headers, out};
use const bad\csrf\CHECK;
use const bad\http\ONE;

return function($args) {
    if (!checkin()) {
        headers(ONE, 'Location', '/login');
        exit(out(302));
    }

    csrf(CHECK, '_csrf') || exit(out(403, 'Invalid token'));

    $order = [
        'user'   => checkin(),
        'symbol' => $_POST['symbol'],
        'qty'    => (int)$_POST['quantity'],
        'price'  => (float)$_POST['price'],
        'type'   => $_POST['type']
    ];

    return trans(function($pdo) use ($order) {
        qp("INSERT INTO orders (user_id, symbol, qty, price, type) VALUES (?, ?, ?, ?, ?)",
            [$order['user'], $order['symbol'], $order['qty'], $order['price'], $order['type']],
            [],
            $pdo);

        return ['order_id' => $pdo->lastInsertId(), 'status' => 'pending'];
    });
};

Scope Guidelines

Sweet Spot:

  • 1–10 developers
  • Performance-critical
  • Direct SQL preferred
  • Custom logic heavy

Manageable:

  • Up to 50,000 LOC
  • High-traffic read-heavy
  • Integration-heavy
